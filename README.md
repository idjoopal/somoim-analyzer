# 다감노📸 소모임 분석

[소모임](https://www.somoim.co.kr) **다감노** 모임의 게시글·사진·멤버를 한 해(또는 특정 월) 단위로 모아 출사·사진·테마 활동·참석 추적 인사이트를 보여주고, 한 파일짜리 종합 통계 엑셀을 내려받는 Streamlit 웹 앱입니다. 년/월만 고르면 되고 개발 지식은 필요 없습니다.

## 주요 기능

- **자동 수집 또는 분석 엑셀 업로드** — somoim API(`/api/articles`·`/api/photos`·`/api/group`)에서 게시글·사진·활성 멤버·가입인사를 모으거나, 이전에 받은 분석 엑셀을 그대로 올리면 API 호출 없이 즉시 같은 분석을 다시 봅니다.
- **🪪 가입인사 자동 매핑 (빠른 수집)** — `/api/articles`에 `cat=J`를 서버 측 필터로 넘겨 가입인사만 받아오고, 활성 멤버 전원의 가입인사를 확보하는 즉시 페이지네이션을 종료합니다. 매칭은 닉네임이 아니라 **user id(게시글 `wid` ↔ 멤버 `mid`)** 기준이라 동명이인이나 닉네임 변경도 안전하게 잡습니다. 본문의 `이름 : XXX`(또는 `성함`/`본명`) 패턴을 그룹 시작부터 스캔해 (실명 → 활성 멤버 닉네임) 매핑을 자동 구축 — 미매칭 이름 해소 작업량을 크게 절약해 줍니다.
- **활성 멤버 기준 집계** — 출사 공지 작성자·사진 업로더·테마사진 참여자 등 **작성자 랭킹은 모두 활성 멤버만** 집계해 탈퇴 멤버가 랭킹을 흐리지 않습니다. 단, **탈퇴 멤버가 연 출사의 후기 참석자 추적은 그대로 유지** — 활성 멤버의 참석 이력은 어디서도 잃지 않습니다.
- **3단계 보정 워크플로**
  1. **① 미매칭 이름 정리** — 후기 본문에서 추출됐지만 활성 멤버 명단과 정확히 일치하지 않는 이름을 드롭다운(마스터 닉네임 매핑 · 🚪 탈퇴 멤버)으로 해소. 이름이 아닌 단어는 **`이름 ❌` 체크박스**로 한 번에 빠르게 거를 수 있습니다. 가입인사 자동 매핑은 기본값으로 미리 채워지고, 사용자가 다른 값을 고르면 사용자 매핑이 우선합니다.
  2. **② 분류·참석자 보정** — 자동 분류가 애매한 공지(출사일 추론 실패·카테고리 미상)와 자동 추출이 부족한 후기 참석자를 `st.data_editor`로 직접 수정.
  3. **③ 인사이트 + 엑셀** — 10개 탭으로 결과를 보고, 사이드바에서 한 파일짜리 엑셀과 매핑 CSV를 내려받습니다.
- **⚠️ 동명이인 마킹** — 같은 닉네임(`mn`)을 가진 활성 멤버가 둘 이상이면 ① 단계 상단 경고, 🧑‍🤝‍🧑 멤버 탭의 metric/전용 테이블, 엑셀 멤버 현황 시트에서 마킹. 후기 본문은 닉네임만 담기 때문에 두 명이 합쳐 집계됨을 사용자가 인지하도록 합니다.
- **인사이트 대시보드 (10개 탭)**
  - 📊 **개요** — 진행/취소·카테고리 도넛, 월별 활동 추이, 핵심 숫자
  - 📌 **출사** — 월별 공지, 작성자 순위, 취소(펑) 순위, 공지 목록
  - 📝 **후기** — 월별 후기 카드, 매핑 적용된 참석자 명단, 매칭된 출사 정보
  - 👥 **참석** — 후기 본문 기반 실제 참석자 매칭, 멤버별·월별 매트릭스, 미매칭 후기
  - 📷 **사진** — 업로드 순위, 월별 업로드, 인기 사진 갤러리
  - 🎨 **테마사진** — 월별 제출 인원 + 그 달 사진 미리보기, 매트릭스, 참여자 순위
  - 🏷️ **카테고리** — 출사 공지의 카테고리 태그 분포
  - 👤 **사용자** — 작성자별 게시글·사진 종합 랭킹
  - 🧑‍🤝‍🧑 **멤버** — 활성 멤버 현황, 유령/휴면 분류, 동명이인 마킹, 월별 신규 가입
  - 📋 **데이터** — 원본 게시글·사진 표 + CSV 다운로드
- **분석 엑셀 = 결과 + 원본 + 매핑** — 12개 가시 시트(인사이트)와 6개 숨김 시트(원본 게시글·사진·활성 멤버·탈퇴 멤버·사용자 이름 매핑·가입인사 자동 매핑)를 한 파일에 담아, 다시 업로드하면 API 호출 0회로 동일 분석을 재현합니다.
- **이름 매핑 CSV** — 가입인사 자동 매핑으로 잡지 못한 닉네임 변형(예: `음승구→승구`)·탈퇴 멤버·노이즈 매핑을 사이드바에서 CSV로 저장/불러오기. 매월 새 데이터로 분석해도 같은 사용자 매핑을 재사용할 수 있습니다.

## 사용법

### 배포된 앱
1. 사이드바에서 **API 수집** 선택 → 년도(필요 시 월) → **분석 시작**.
   - 또는 이전에 받은 **분석 엑셀**을 업로드하면 API 호출 없이 같은 분석을 즉시 다시 봅니다.
2. **① 미매칭 이름 정리** — 자동 매핑으로 미리 채워진 드롭다운을 그대로 두거나 수정. 이름이 아닌 토큰은 **`이름 ❌`** 체크박스 컬럼에서 위에서 아래로 한 번에 클릭해 빠르게 제외하세요.
3. **② 분류·참석자 보정** — 검토가 필요한 공지/후기만 손보고 **이 분류·참석자로 분석 진행**.
4. **③ 인사이트** — 10개 탭을 둘러보고, 사이드바의 **💾 엑셀 다운로드** + **🧭 이름 매핑 CSV 저장**으로 결과 보관.

> 분석이 끝나면 엑셀 한 파일만 받으면 됩니다. 다음에 그 엑셀을 다시 올리면 API 호출 없이 같은 분석이 열려요. 다른 달/년 분석에는 **이름 매핑 CSV**만 들고 가면 사용자 매핑을 재사용할 수 있습니다.

### 로컬 실행
```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

### 테스트
```bash
python -m pytest tests/ -q
```
순수 함수 단위 테스트(이름 추출·해소·가입인사 매핑·동명이인·매칭·엑셀 라운드트립 등)는 네트워크 없이 모두 통과해야 합니다.

### 코드로 직접 (선택)
```python
from core import (
    collect_posts, collect_photos,
    collect_members, collect_banned_names,
    collect_join_greetings, parse_join_name_aliases,
    annotate_attendees, match_outings_with_reviews,
    save_excel,
)

year = 2026
posts  = collect_posts(year)       # 특정 월만: month=5
photos = collect_photos(year)
members, master = collect_members()
banned = collect_banned_names()

# 가입인사: cat=J 서버 필터 + 활성 멤버 mid 기반 조기 종료
min_joined = min((m["joined_at"] for m in members if m.get("joined_at")), default=None)
joins = collect_join_greetings(active_members=members, min_joined_at=min_joined)
join_aliases = parse_join_name_aliases(joins, active_mns=master)

annotate_attendees(posts, master, join_aliases)
match_outings_with_reviews(posts)
save_excel(posts, photos, year,
           members=members, banned=banned,
           resolution={}, join_aliases=join_aliases)
# → 다감노_2026_분석.xlsx 저장
```

## 배포 (Streamlit Community Cloud · 무료)

서버 비용 0원이며 `main`에 push하면 자동 재배포됩니다. **Deploy 클릭은 본인 GitHub/Streamlit 계정**으로 진행해야 합니다(대행 불가).

1. https://share.streamlit.io → **Sign in** (GitHub 로그인·권한 인가)
2. **Create app** → **Deploy a public app from GitHub**
3. 입력
   - Repository: `idjoopal/somoim-analyzer`
   - Branch: `main`
   - Main file path: `streamlit_app.py`
   - App URL: 원하는 서브도메인
4. (선택) Advanced settings → Python 3.11 이상
5. **Deploy** → 1~3분 빌드 후 `https://<서브도메인>.streamlit.app` 발급 → 공유

운영 중에는 앱 우측 하단 **Manage app**에서 빌드·런타임 로그를 볼 수 있습니다. 일정 시간 미접속 시 sleep 되고, 다음 접속 때 수십 초 내 자동 기동됩니다.

## 집계 기준

- **기간** — 출사 공지(`cat=A`)는 **출사일** 기준, 후기·가입인사·사진은 **작성일** 기준.
- **활성 vs 비활성** — 작성자 랭킹(공지·후기·사진·테마사진)은 활성 멤버(`/api/group`의 `ban=N`)만 집계. 후기 본문에서 추출되는 참석자 추적은 **공지 작성자가 탈퇴했어도 유지** — 활성 멤버의 참석 이력 손실을 막기 위함.
- **카테고리** (제목 `[태그]`) — 출사: 인물(1:1인물·1:1인물출사 포함)·인물&풍경(구 인풍)·풍경·GN / 활동: 보정·문화.
- **취소(펑)** — 제목에 `(펑)`·`[펑]` 포함 시 취소로 집계.
- **참석자** — 후기 본문에서 한글 2~4자·영문 2~10자 토큰을 추출 → 활성 멤버 닉네임(`mn`) 매칭 → 가입인사 자동 매핑 → 사용자 매핑(또는 🚪 탈퇴 / `이름 ❌` 노이즈) 순으로 해소. 매칭된 출사 공지의 참석자로 부착됩니다.
- **출사 매칭** — 후기의 제목/본문에서 추정한 출사일(없으면 작성일)로 가까운 공지를 찾고, 카테고리·작성자 일치 시 가산점.
- **테마사진** — 댓글이 달린 사진(`rn>0`)을 테마 참여로 **추정**합니다. 댓글 내용은 비공개라 개수만 활용하므로, 테마사진 탭의 월별 미리보기로 실제 여부를 눈으로 확인할 수 있습니다.
- **인기** — 좋아요 수로 정렬하고 댓글 수를 함께 표기.
- **동명이인** — 같은 닉네임 활성 멤버가 둘 이상이면 후기 본문에선 분리 불가 → 한 명으로 합쳐 집계됨을 ⚠️로 마킹.

## 주의사항

- **비공식 API 기반** — 소모임 응답 구조나 인증 정책이 바뀌면 수집이 멈출 수 있어 정기적인 동작 확인이 필요합니다. 댓글 **내용**은 인증 한계로 가져오지 않습니다(개수만 활용).
- **타임스탬프 보정값**(`+1_000_000_000`)은 실측 추정값이라, 큰 오차가 보이면 재추정이 필요합니다.
- **사진 CDN URL**은 현재 무기한이지만 만료될 수 있으니 필요한 사진은 즉시 보관을 권장합니다.
- **`/api/articles` 페이지 한도** — 200페이지 한도가 있지만 `cat=J` 서버 필터로 가입인사만 받고 활성 멤버 전원 확보 시 조기 종료하므로 실제로 한도에 닿을 일은 거의 없습니다.
- 분석 대상은 **다감노 모임 한 곳**(`GROUP_ID` 하드코딩)이며 **한 해 단위**입니다.
- **이용약관** — 자동 수집이 소모임 약관과 충돌할 수 있으므로 개인 분석 용도로만 사용하세요.
