# 다감노📸 소모임 분석

[소모임](https://www.somoim.co.kr) **다감노** 모임의 게시글·사진·멤버를 한 해(또는 특정 월) 단위로 모아 출사·사진·테마 활동·참석 추적 인사이트를 보여주고, 한 파일짜리 종합 통계 엑셀을 내려받는 Streamlit 웹 앱입니다. 년/월만 고르면 되고 개발 지식은 필요 없습니다.

## 수집·분석 파이프라인

```
somoim API
 ├─ /api/articles  → 공지(cat=A) · 후기(cat=E) · 가입인사(cat=J, 서버 측 cat 필터)
 ├─ /api/photos    → 사진 (CDN URL 자동 생성)
 └─ /api/group     → 활성/탈퇴 멤버 (mid · 닉네임 · 가입일 · 마지막 방문)
        ↓
  ① 미매칭 이름 정리 (Stage 1)  — 가입인사 자동 매핑 + 사용자 보정
        ↓
  ② 분류 · 참석자 보정 (Stage 2) — 출사일/카테고리 미상 공지, 후기 참석자 수동 편집
        ↓
  ③ 10개 탭 인사이트 (Stage 3) + 한 파일짜리 엑셀
```

- **가입인사 수집은 cat=J 서버 필터** + 활성 멤버 mid 기반 조기 종료로 그룹 시작까지 거슬러 가도 빠릅니다.
- **활성/탈퇴 구분은 user id(`wid`↔`mid`) 기준** — 닉네임 일치가 아니라 user id로 매칭해 동명이인·닉네임 변경에도 흔들리지 않습니다.
- **작성자 랭킹은 활성 멤버만** 집계하지만, **탈퇴 멤버가 연 출사 후기의 참석자 추적은 그대로 유지** — 활성 멤버의 참석 이력은 어디서도 잃지 않습니다.

## 사용법

### 배포된 앱
1. **사이드바 → API 수집** 선택 → 년도(필요 시 월) → **분석 시작**
   - 또는 이전에 받은 **분석 엑셀**을 업로드 → API 호출 0회로 같은 분석을 즉시 다시 봅니다.
2. **① 미매칭 이름 정리** — 후기 본문에서 추출된 이름 중 활성 멤버 명단과 정확히 일치하지 않는 것을 정리합니다.
   - 드롭다운: 마스터 닉네임으로 매핑 또는 🚪 탈퇴 멤버
   - 이름이 아닌 단어는 **`이름 ❌` 체크박스**로 한 번에 빠르게 제외
   - 가입인사 본문에서 자동 추출된 `이름 : XXX` 매핑은 기본값으로 미리 채워져 있음
3. **② 분류·참석자 보정** — 검토가 필요한 공지(출사일 추론 실패·카테고리 미상)와 후기 참석자만 손보고 다음 단계. 후기 본문은 전체가 셀에 들어 있어 클릭하면 펼쳐 볼 수 있습니다.
4. **③ 인사이트** — 아래 10개 탭으로 결과를 확인하고, 사이드바 **💾 엑셀 다운로드** + **🧭 이름 매핑 CSV 저장**으로 결과 보관.

### 로컬 실행
```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

### 테스트
```bash
python -m pytest tests/ -q
```
순수 함수 단위 테스트(이름 추출·해소·가입인사 매핑·동명이인·매칭·엑셀 라운드트립 등)는 네트워크 없이 모두 통과해야 합니다.

| 📌 **출사** | 월별 진행/취소 막대, **출사 공지 작성 순위** (활성 멤버), **취소(펑) 순위** (공지 3건+), 공지 전체 목록 |

| 📷 **사진** | 사진 업로드 순위(활성), 월별 업로드, 인기 사진 갤러리(좋아요 TOP 12) |
| 🎨 **테마사진** | 월별 제출 인원, 월별 expander로 참여자 + 그 달 사진 미리보기, 작성자×월 매트릭스, 참여자 순위 |
| 🏷️ **카테고리** | 출사 공지 제목 `[태그]` 기준 분포 (가로 막대 + 표) |
| 👤 **사용자** | 작성자별 게시글·사진 종합 랭킹 (활성 멤버) |
| 🧑‍🤝‍🧑 **멤버** | 활성 멤버 수·운영진·iOS/Android·유령 멤버·⚠️ 동명이인 metric, 동명이인/유령 멤버 표, 월별 신규 가입 |
| 📋 **데이터** | 원본 게시글·사진 데이터프레임 + CSV 다운로드 |

## 분석 엑셀 (한 파일에 결과 + 원본 + 매핑)

가시 시트 12개 (+ 멤버 정보 있으면 13개) — UI 인사이트를 그대로 옮긴 결과:

1. **📊 대시보드** — KPI 6장 + 월별 활동 표 + 진행/취소 클러스터드 바 차트
2. **👤 게시글 통계** — 작성자별 공지/취소/후기/카테고리별/좋아요/취소율 + 데이터바
3. **📌 출사 공지** — 공지 전체 목록 (취소·진행 색상 구분)
4. **📝 후기글** — 후기 전체 목록
5. **📷 사진** — 사진 전체 목록 (테마사진 색상 강조)
6. **🎨 월별 테마 매트릭스** — 작성자 × 월
7. **👤 사진 통계** — 작성자별 사진 수·테마 비율·좋아요
8. **🏷️ 카테고리** — 카테고리 × (공지 수·진행·취소·좋아요 합) + 데이터바
9. **💡 인사이트** — ① 활동 규모, ② 핵심 기여자 TOP 3, ③ 사진 TOP 5, ④ 테마 패턴, ⑤ 카테고리 트렌드, ⑥ 취소율 분석
10. **🎯 출사별 참석자** — UI 👥 참석 탭과 동일: 공지 제목/후기 제목 분리, `✓ 매칭` / `⚠️ 후기 없음` 상태, 취소 제외 + **고아 후기 섹션** (공지 매칭 안 된 후기)
11. **👥 멤버별 참석** — 멤버 × 카테고리 매트릭스 + 데이터바
12. **📅 월별 참석 매트릭스** — 멤버 × 월 (참석=▣)
13. **🧑‍🤝‍🧑 멤버 현황** — 활성/휴면/유령 분류, 동명이인 ⚠️ 마킹 (members 수집 시)

**숨김 시트** — 재업로드 시 API 호출 0회로 동일 분석을 재현하기 위한 원본 데이터:
`_메타` · `_원본_게시글` · `_원본_사진` · `_멤버마스터` · `_멤버` · `_탈퇴멤버` · `_이름매핑` · `_가입인사매핑`

> 분석이 끝나면 엑셀 한 파일만 받으면 됩니다. 다음에 그 엑셀을 다시 올리면 API 호출 없이 같은 분석이 열려요. 다른 달/년 분석에는 **이름 매핑 CSV**만 들고 가면 사용자 매핑을 재사용할 수 있습니다.

### 코드로 직접 (선택)

```python
from core import (
    collect_posts, collect_photos,
    collect_members, collect_banned_names,
    collect_join_greetings, parse_join_name_aliases,
    annotate_attendees, match_outings_with_reviews,
    save_excel,
)

year = 2026
posts  = collect_posts(year)       # 특정 월만: month=5
photos = collect_photos(year)
members, master = collect_members()
banned = collect_banned_names()

# cat=J 서버 필터 + 활성 멤버 mid 기반 조기 종료
min_joined = min((m["joined_at"] for m in members if m.get("joined_at")), default=None)
joins = collect_join_greetings(active_members=members, min_joined_at=min_joined)
join_aliases = parse_join_name_aliases(joins, active_mns=master)

annotate_attendees(posts, master, join_aliases)
match_outings_with_reviews(posts)
save_excel(posts, photos, year,
           members=members, banned=banned,
           resolution={}, join_aliases=join_aliases)
# → 다감노_2026_분석.xlsx 저장
```

## 집계 기준

- **기간** — 출사 공지(`cat=A`)는 **출사일** 기준, 후기·가입인사·사진은 **작성일** 기준.
- **활성 vs 비활성** — 작성자/업로더 랭킹(공지·후기·사진·테마사진)은 활성 멤버(`/api/group`의 `ban=N`)만 집계. 후기 본문에서 추출되는 참석자 추적은 **공지 작성자가 탈퇴했어도 유지**.
- **카테고리** (제목 `[태그]`) — 출사: 인물(1:1인물·1:1인물출사 포함)·인물&풍경(구 인풍)·풍경·GN / 활동: 보정·문화.
- **취소(펑)** — 제목에 `(펑)`·`[펑]` 포함 시 취소로 집계. 출사별 참석자 표·🎯 출사별 참석자 시트에서는 제외.
- **참석자** — 후기 본문에서 한글 2~4자·영문 2~10자 토큰을 추출 → 활성 멤버 닉네임(`mn`) 매칭 → 가입인사 자동 매핑 → 사용자 매핑(또는 🚪 탈퇴 / `이름 ❌` 노이즈) 순으로 해소. 매칭된 출사 공지의 참석자로 부착됩니다.
- **출사 ↔ 후기 매칭** — 후기 제목/본문에서 추정한 출사일(없으면 작성일)로 거리 ≤ ±7일(추정 성공) / ±45일(fallback) 내 공지 후보를 찾고, 카테고리 일치 시 -100, 작성자 일치 시 -5 가산점. 거리 가까운 후기부터 공지를 선점해 1:1 매칭. 제목 단어는 매칭에 사용하지 않습니다.
- **테마사진** — 댓글이 달린 사진(`rn>0`)을 테마 참여로 **추정**합니다. 댓글 내용은 비공개라 개수만 활용하므로, 🎨 테마사진 탭의 월별 미리보기로 실제 여부를 눈으로 확인할 수 있습니다.
- **인기** — 좋아요 수로 정렬하고 댓글 수를 함께 표기.
- **동명이인** — 같은 닉네임 활성 멤버가 둘 이상이면 후기 본문에선 분리 불가 → 한 명으로 합쳐 집계됨을 ⚠️로 마킹.

## 배포 (Streamlit Community Cloud · 무료)

서버 비용 0원이며 `main`에 push하면 자동 재배포됩니다. **Deploy 클릭은 본인 GitHub/Streamlit 계정**으로 진행해야 합니다(대행 불가).

1. https://share.streamlit.io → **Sign in** (GitHub 로그인·권한 인가)
2. **Create app** → **Deploy a public app from GitHub**
3. 입력
   - Repository: `idjoopal/somoim-analyzer`
   - Branch: `main`
   - Main file path: `streamlit_app.py`
   - App URL: 원하는 서브도메인
4. (선택) Advanced settings → Python 3.11 이상
5. **Deploy** → 1~3분 빌드 후 `https://<서브도메인>.streamlit.app` 발급 → 공유

운영 중에는 앱 우측 하단 **Manage app**에서 빌드·런타임 로그를 볼 수 있습니다. 일정 시간 미접속 시 sleep 되고, 다음 접속 때 수십 초 내 자동 기동됩니다.

## 주의사항

- **비공식 API 기반** — 소모임 응답 구조나 인증 정책이 바뀌면 수집이 멈출 수 있어 정기적인 동작 확인이 필요합니다. 댓글 **내용**은 인증 한계로 가져오지 않습니다(개수만 활용).
- **타임스탬프 보정값**(`+1_000_000_000`)은 실측 추정값이라, 큰 오차가 보이면 재추정이 필요합니다.
- **사진 CDN URL**은 현재 무기한이지만 만료될 수 있으니 필요한 사진은 즉시 보관을 권장합니다.
- **`/api/articles` 페이지 한도** — 200페이지 한도가 있지만 `cat=J` 서버 필터로 가입인사만 받고 활성 멤버 전원 확보 시 조기 종료하므로 실제로 한도에 닿을 일은 거의 없습니다.
- 분석 대상은 **다감노 모임 한 곳**(`GROUP_ID` 하드코딩)이며 **한 해 단위**입니다.
- **이용약관** — 자동 수집이 소모임 약관과 충돌할 수 있으므로 개인 분석 용도로만 사용하세요.
